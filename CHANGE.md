  ## 목표                                                                                                                                                          
  사용성에 대한 고려를 늘리고 기능적인 측면을 축소(MVP 축소)                                                                                                             
  ## 주요 변경                                                                                                                                                          
  - 일단 과업이라는 개념에만 집중, 장기과업(Goal) 개념은 일단 무시                                                                                                       
  - 디자인 프린시플: 모바일 환경에서 PWA 앱을 사용한다고 가정하고, 최대한 사용자가 포커스를 옮길 필요 없도록 구현. 접근성도 최대한으로 고려. 폼의 경우 Progressive       
  Form(혹은 Auto-Advance Form) 형태로 구현하여 사용자가 단계별 스텝을 쉽게 따라갈 수 있도록 고려.                                                                        
  - 항상 앱 하단에 Floating 채팅 Input 을 표기. 가급적이면 모든 입력은 그곳을 통해서 진행.                                                                               
  - 계획과 회고 단계 삭제. 그냥 해당 일자에 계획이 없으면 계획 페이지로 네비게이션. 회고 단계 역시도 단순한 로그 보기 화면으로 변경.          
  ## 주요 페이지 기획 초안                                                                                                                                                   
  ### 계획 페이지
  기본적으로 Progressive 한 형태로, 계속해서 내용을 쌓아가며 아래로 스크롤하며 진행
  #### 엔트리
  - 해당 날짜에 만든 계획이 없다면 인사와 함께 계획을 시작하시겠습니까 + 시작하기 버튼 출력
  #### 어제 완료하지 못한 과업
  - 시작하면 전날 완료하지 못한 과업 리스트 보여주기. 완료하지 못한 과업이 없으면 없다고 출력
  - 과업 리스트의 과업들을 모두 비활성화(혹은 선택되지 않은 상태)로 출력.
  - 오늘 이어서 진행할 과업을 선택하라는 안내 문구와 함께, 각 과업을 클릭하면 오늘 과업에 포함/미포함 여부가 토글
  - 진행하기 버튼 출력하고, 진행하기 버튼을 누르면 선택된 과업들을 오늘 수행할 과업으로 포함시켜서 다음 단계로 진행
  #### 과업 추가하기
  - 오늘 작업할 과업들을 추가하는 단계
  - 전체 과업 리스트를 항상 보여주고 있고, 플로팅 인풋을 통해서 과업 내용을 추가. 
  - 리스트에서 수정 혹은 삭제 버튼 사용 가능
  - 수정의 경우, 인라인에서 수정하는 듯한 형태로 스타일링
  - 마찬가지로 완료 버튼을 누르면 검토 단계로 이동
  #### 과업 검토하기
  - AI 가 과업을 검토해서 더 작게 나누거나 하는 변경을 제안.
  - 수락하든 거절하든 넘어가면 과업 페이지로 이동.

  ### 과업 페이지
  - 전체 과업이 리스트로 노출
  - 최상단에는 현재 진행중인 과업을 위한 별도 공간이 위치하고, 거기에 진행중 과업 표시
  - 각 과업에는 체크박스가 붙어있고, 체크하면 완료처리 되며 전체 리스트의 최 하단으로 이동. 
  - 진행중이지 않고 대기중인 과업을 누르면 진행중 과업을 교체. 기존 진행중 과업은 그냥 대기중 과업으로 변경
  - 모든 과업의 상태 변경은 각 과업에 로그를 남김
  - 페이지 하단에 플로팅 인풋을 통해서 새로 과업을 생성하거나 AI 와 함께 수정/삭제를 진행할 수 있음. 
  
  ### 로그 페이지
  - 각 과업을 타임라인에 맞추어 언제 진행했었는지 표시

  ### 설정 페이지
  - AI API 키를 관리
